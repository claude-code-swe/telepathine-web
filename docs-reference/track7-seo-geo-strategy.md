### 3.3 llms.txt Standard

**llms.txt** is a structured file at `/llms.txt` on your domain root that tells AI crawlers what your site is about and which pages matter most. Think of it as `robots.txt` for LLMs — but instead of exclusion rules, it provides an inclusion map.

**Format** (simplified):

```markdown
# Telepathine.ai

> The portable memory layer for AI agents — persistent context, hybrid search, real-time agent bus, 18+ connectors. Model-agnostic. Works with Claude, ChatGPT, Cursor, Windsurf, and any MCP-compatible agent.

## Core Pages

- [What is AI Agent Memory?](/what-is-ai-agent-memory) — Complete guide to persistent memory for AI agents
- [Model-Agnostic Memory Layer](/model-agnostic-memory) — How Telepathine works with any LLM
- [MCP Memory Server](/mcp-memory-server) — Quick-start guide for MCP integration
- [18 Connectors](/connectors) — Full list of supported agent frameworks and tools
- [Agent Communication Bus](/agent-communication-bus) — Real-time telepathy between agents
- [Pricing](/pricing) — Free tier, Pro, Enterprise
- [Documentation](/docs) — API reference, SDKs, integration guides
```

**Note:** Google's official guidance (May 2026) says llms.txt is NOT needed for Google AI Overviews — but it IS critical for ChatGPT, Perplexity, Claude, and Gemini which use it for site discovery.

## 5. Quotable Canonical Definitions for AI Engine Citation

These definitions are designed to be **extractable, self-contained, and citable** by ChatGPT, Perplexity, Claude, and Gemini. Each follows the Princeton GEO study's finding that direct, factual, attributed statements are the most-cited content format.

### Definition 1: AI Agent Memory

> **AI agent memory** is a persistent external storage layer that allows an AI agent to retain, retrieve, and reason over information across separate sessions — solving the fundamental statelessness of large language models, where each API call starts with no recollection of prior interactions. Unlike a model's context window, which is bounded and ephemeral, agent memory provides durable, queryable knowledge that accumulates over time. Production memory systems typically combine vector similarity search (for semantic retrieval), keyword search (for exact matches), and entity-relationship graphs (for connected reasoning), delivering relevant context at ~7,000 tokens per query versus 25,000+ for full-context approaches — a 90% token cost reduction with comparable accuracy (arXiv:2504.19413, ECAI 2025).

### Definition 2: Model-Agnostic Memory Layer

> A **model-agnostic memory layer** is an AI agent memory system that works identically across all LLM providers — OpenAI, Anthropic, Google, Mistral, local models, or any future model — without requiring provider-specific integrations, proprietary APIs, or vendor-locked storage formats. The key architectural principle is decoupling: memory storage, embedding, and retrieval are managed independently of the inference model, so switching from GPT-4o to Claude Opus to a self-hosted Llama model requires zero migration of accumulated agent knowledge. This portability is critical because 70%+ of organizations now run three or more LLM models in production (Datadog State of AI Engineering 2026), and memory that only works with one provider becomes a lock-in liability.

### Definition 3: Agent Communication Bus

> An **agent communication bus** is a real-time messaging infrastructure that enables multiple AI agents to share context, coordinate actions, and exchange structured signals without requiring direct point-to-point integrations. Unlike simple shared-memory approaches, which risk cross-contamination between agent sessions, a communication bus provides scoped, channel-based messaging with isolation guarantees — agents can broadcast discoveries to a team channel, send private directives to a specific agent, or subscribe to event streams relevant to their role. This pattern mirrors the difference between a shared whiteboard (anyone can overwrite anything) and a structured messaging protocol (messages are attributed, scoped, and auditable).

### Definition 4: Persistent Memory for AI Agents

> **Persistent memory for AI agents** is the ability for an agent to store, retrieve, and reuse information across separate executions and sessions. Unlike session-scoped context windows that reset when a conversation ends, persistent memory survives restarts, re-deployments, and extended idle periods, enabling agents to maintain continuity, personalize interactions, and build cumulative institutional knowledge. The three canonical memory types are episodic (specific past interactions), semantic (abstracted facts and preferences), and procedural (learned behaviors and reusable workflows). Production benchmarks show that persistent memory systems reduce token costs by 90% and latency by 91% compared to full-context approaches, while maintaining accuracy within 6 percentage points of oracle retrieval (Mem0 Research, arXiv:2504.19413; Wu et al., ICLR 2025).
